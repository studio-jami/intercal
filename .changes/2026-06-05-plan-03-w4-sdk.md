# Plan 03 W4 — TypeScript SDK over the V1 REST surface

Date: 2026-06-05
Type: feat
Packages: @intercal/sdk

## Summary

Built a complete, fully-typed `@intercal/sdk` client over the live V1 REST surface
(`/api/v1/*`). Replaces the scaffold with real request building, an injectable fetch, safe
retries, and a typed error model that mirrors the REST error taxonomy. Every request param and
response type is derived from the generated contract (`@intercal/shared`) — no hand-redeclared
shapes, single source of truth. The two deferred operations (`getDelta` W5, `verifyClaim` W6)
are real typed methods that surface the live `501 not_implemented` cleanly, not faked.

## Changes

- **Contract-derived types.** `Query<Op>` / `Ok<Op>` conditional types extract each operation's
  query params and 200 response from the generated `operations` interface, so the public param
  and response aliases (`EntityParams`, `EntityResponse`, …) are projections of the contract. A
  contract change propagates into the SDK at compile time.
- **Six methods.** `getEntity`, `getSources`, `getFreshness`, `searchEvidence`, `getDelta`,
  `verifyClaim` — one per contract operation, routed to the TypeSpec paths
  (`/v1/claims/verify`, not a REST-ism). `token_budget` is in the delta/verify signatures per
  the contract (server applies it once the body lands in W5/W6).
- **Typed error model.** Base `IntercalApiError` (discriminant `code`, plus `status` +
  `details`) with `instanceof`-discriminable subclasses: `IntercalInvalidRequestError` (400),
  `IntercalNotFoundError` (404), `IntercalNotImplementedError` (501), `IntercalServerError`
  (500), and `IntercalNetworkError` (status 0) for transport failures. `errorFor` maps the
  served `ApiError.code` to the right class; unknown codes fall back to the base class.
- **Transport.** Base URL normalization, query assembly (undefined params omitted), injectable
  `fetch` (bound to `globalThis`), bearer `apiKey` (Plan 04 auth seam), extra headers, and
  GET-only retries with exponential backoff for transient network/5xx failures — 4xx and 501
  are deterministic and never retried. No pagination helper: the V1 contract exposes no
  cursor/offset surface, so one would be a placeholder; deferred until the contract grows one.

## Tests

- `packages/sdk/src/index.test.ts` (14) — fixture-backed contract tests with an injected,
  request-recording fetch: URL/query building, header + bearer auth, typed responses returned
  unchanged, the full error-taxonomy mapping (400/404/501/500 + unmapped + network), and retry
  behavior (retries 5xx, never retries 501).
- `packages/sdk/src/fixtures.ts` — real responses captured from the live surface against
  production Neon, typed against the generated contract (a contract change breaks compilation
  here — a typed tripwire, not faked product data).
- `packages/sdk/src/live.test.ts` (5, opt-in via `INTERCAL_LIVE=1`) — smoke test against
  `https://lntercal.vercel.app/api/v1/*`: real `getEntity` / `searchEvidence` / `getFreshness`
  reads, and `getDelta` / `verifyClaim` surfacing typed `IntercalNotImplementedError` (501).
